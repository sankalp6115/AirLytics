Working of our system

    Data Acquisiton

  • Air Data: Fetched from OpenWeatherMap API using city name or geolocation (latitude/longitude).
  • Weather Data: Temperature, humidity, and weather condition.
  • Air Pollution Data: AQI and pollutant concentrations (PM2.5, PM10, NO2, O3).
  • User Input: Age, gender, pincode, and symptoms.

    • Weighted Risk Factor Calculation

    Weighted Risk Factor Calculation Weights assigned per pollutant like AQI has 0.35 weight, PM2.5 has 0.25 weight and so on. Score Mapping: Each pollutant scored 0–3 based on thresholds. Overall Risk Score: 
    
    totalScore = Σ(score_i × weight_i)
    normalizedRisk = totalScore / Σ(weights)
  • Risk Level Assignment: 
                0 – 0.5 → Low
            0.5 – 1.5 → Moderate
            1.5 – 2.5 → High
            2.5 – 3 → Severe

    • Visual Representation

  • Gauge/Needle for AQI
    • Map AQI (0–500) to rotation angle (-90° to +90°).
    Formula: angle = ((AQI - min) / (max - min)) * 180 - 90
  • Color Coding: 
    • Green → Good, Orange → Moderate, Red → Bad, Maroon → Very Bad
    Applied to DOM elements for visual clarity.

    PDF report Generation

  • Right now the self-diagnosis form completed by user, is being returned in a PDF Report format. But in the future, we seek to store this data securely in a database for using this data for ML prediction.
